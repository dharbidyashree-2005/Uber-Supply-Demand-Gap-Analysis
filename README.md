# Uber Supply Demand Gap Analysis

## Project Overview

This project analyzes the supply-demand gap in Uber ride requests using Data Analytics techniques. The objective is to identify the reasons behind ride cancellations and unavailability of cabs, and provide business recommendations to improve customer satisfaction.

## Problem Statement

Customers often face ride cancellations or "No Cars Available" messages during certain hours of the day. This project aims to identify the patterns behind these issues and understand the demand-supply imbalance.

## Tools Used

- Excel (Data Cleaning & Dashboard)
- SQL (Business Insights)
- Python (Pandas, NumPy, Matplotlib)
- Google Colab / Jupyter Notebook

## Dataset Features

- Request ID
- Pickup Point
- Driver ID
- Status
- Request Timestamp
- Drop Timestamp

## Key Findings

- Highest supply-demand gap occurs during Night and Early Morning.
- Airport requests face maximum cab unavailability.
- Driver cancellations are higher during Morning hours.
- Better driver allocation and surge pricing can reduce the gap.

## Business Recommendations

- Introduce incentives for drivers during peak hours.
- Increase driver availability during night shifts.
- Improve demand forecasting and resource allocation.
